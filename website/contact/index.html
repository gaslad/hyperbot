Contact

Reach the project.

Use the repository and issue tracker as the primary contact path for Hyperbot. That keeps support requests public, searchable, and attached to the codebase instead of disappearing into a one-off form.

Primary contact path

Project repository

Open issues for bugs, documentation gaps, feature requests, or questions about the landing page and trading workflow.

What to include

  • The page or workflow you were using.
  • The behavior you expected.
  • Any relevant console output, logs, or screenshots.

Why this page matters

A visible contact route is a basic authority signal. It also gives the homepage a real inbound link once the rest of the trust pages are in place, which improves crawlability and avoids the single-page dead-end pattern.
